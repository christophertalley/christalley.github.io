EZforms

EZforms was built using a react frontendand an express backend with a MongoDB database. In EZforms, Admins can create and customize forms, share forms to be submitted, and analyze the results. EZforms is special to me, because it was one of the toughest projects I worked on.

I chose a non-relational database due to the fact that I wanted to be able to dynamically handle form creations and submissions. Mongodb was by far the best technology for the job. On form creation, a data sctructure was formed based off of what fields were added and customized, and then MongoDB handled all of the created forms in the same collection. When a form is loaded, the same data is fetched and a form is rendered based off of the returned data.

Users on EZforms were authenticated using Auth0 and also stored in the MongoDB database. Each user has access to only forms they created, but anyone is free to access and submit the form once sent the link.

The frontend for EZforms was hosted on AWS Amplify and the backend was hosted using Heroku. The live site can be found here.
